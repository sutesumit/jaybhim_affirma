# Sitemap & Metadata ŌĆö Phase 2 Refactoring Plan

## Status: Phase 1 Complete, Phase 2 Partially Done

Phase 1 (initial implementation) is done. Phase 2 reduced `staticPageMetadata` from 9 fields/page to 4 fields/page via `seoOverrides`, but **the core duplication remains**: every page key in `seoOverrides` is still a manual mirror of `pages-registry` keys. The `getProjectSeoMeta()` returns `null` if no override exists, meaning every new page **still requires** editing `config/metadata.ts`.

This plan finishes the deduplication: make `seoOverrides` fully optional with defaults.

## Fix: Make overrides optional with defaults

### New `getProjectSeoMeta()` logic

```typescript
const SEO_DEFAULTS = {
  ogType: 'website' as const,
  schemaKind: 'WebPage' as const,
  sitemapPriority: 0.7,
};

export function getProjectSeoMeta(key: string): StaticPageMeta | null {
  const page = pagesRegistry[key];
  if (!page) return null;

  const override = seoOverrides[key];

  return {
    path: page.slug,
    title: page.title,
    description: page.shortDescription,
    ogTitle: `${page.title} | ${SITE_NAME}`,
    ogDescription: override?.ogDescription ?? page.shortDescription,
    ogType: override?.ogType ?? SEO_DEFAULTS.ogType,
    schemaKind: override?.schemaKind ?? SEO_DEFAULTS.schemaKind,
    ogImage: page.thumbnail,
    sitemapPriority: override?.sitemapPriority ?? SEO_DEFAULTS.sitemapPriority,
  };
}
```

Key change: `if (!page || !seo) return null` ŌåÆ `if (!page) return null`. The override is optional.

### What `seoOverrides` becomes

Only pages that need **non-default** SEO values stay in the map. Everything else is auto-derived from the registry.

Currently all 9 pages have custom `ogDescription` (social card copy that differs from `shortDescription`). After this change:

- Pages where `ogDescription === shortDescription` ŌåÆ remove from `seoOverrides` entirely
- Pages where `ogDescription !== shortDescription` ŌåÆ keep only that one field in `seoOverrides`
- Pages where `ogType !== 'website'` (e.g., `artist-statement` is `'profile'`, `notes-on-atrocity-cases` is `'article'`) ŌåÆ keep `ogType`
- Pages where `schemaKind !== 'WebPage'` (e.g., `documentary-portfolio` is `'CollectionPage'`) ŌåÆ keep `schemaKind`
- Pages where `sitemapPriority !== 0.7` (e.g., `fathers-and-figures` is `0.8`, `at-home-elsewhere` is `0.5`) ŌåÆ keep `sitemapPriority`

### Minimal override structure

Make all fields in `SeoOverride` optional (partial):

```typescript
type SeoOverride = {
  ogDescription?: string;
  ogType?: 'website' | 'article' | 'profile';
  schemaKind?: 'WebPage' | 'ProfilePage' | 'CollectionPage';
  sitemapPriority?: number;
};
```

Pages that differ from defaults keep only the fields they override:

```typescript
export const seoOverrides: Record<string, Partial<SeoOverride>> = {
  'documentary-portfolio': {
    ogDescription: 'A documentary portfolio examining lived realities...',
    schemaKind: 'CollectionPage',
    sitemapPriority: 0.8,
  },
  'fathers-and-figures': {
    ogDescription: 'As I approach my father\'s age...',
    sitemapPriority: 0.8,
  },
  'at-home-elsewhere': {
    ogDescription: 'Imagining friends on the walls...',
    sitemapPriority: 0.5,
  },
  'artist-statement': {
    ogType: 'profile',
    schemaKind: 'ProfilePage',
    sitemapPriority: 0.6,
  },
  'notes-on-atrocity-cases': {
    ogDescription: 'A textual-visual field diary...',
    ogType: 'article',
  },
};
```

Pages like `home-photobook`, `lunchboxes`, `niranjan-in-a-city`, `marathi-short-stories` ŌĆö where all values match defaults ŌĆö get **removed entirely** from `seoOverrides`. They're auto-derived from the registry.
