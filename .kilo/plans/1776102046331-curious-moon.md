# Plan: Sitemap XSLT Stylesheet

## Goal
Create an XSLT stylesheet for `sitemap.xml` that renders a visually styled HTML page in browsers, using the project's design language from `globals.css`.

## Design Tokens (from `globals.css`)
| Token | Value | Usage |
|---|---|---|
| Primary Blue | `#131857` | Headings, accents, borders |
| White | `#ffffff` | Background, text on blue |
| Destructive Red | `#a50c0c` | Warnings / highlights |
| Medium Blue | `#2a3196` | Gradient mid |
| Light Blue | `#455dd6` | Gradient end |
| Near-white | `#f5f5f5` | Alternating row bg |
| Border Gray | `#e5e5e5` | Table borders |
| Muted Blue | `#4a5bbd` | Secondary text |
| Font: Lora | Serif | Headings |
| Font: Merriweather | Serif | Body text |
| Font: Rajdhani | Sans-serif | Labels, badges |
| Radius | `0.5rem` | Cards, buttons |

## Files to Create/Modify

### 1. Create `public/sitemap.xsl`
XSLT stylesheet that transforms sitemap XML into a styled HTML page with:
- **Header**: Site title "Jaybhim Affirma — Sitemap" with primary blue gradient background (matching `.gradient-button` palette)
- **Stats bar**: Total URL count, last modified range
- **Table**: Styled `<table>` with alternating row colors, hover effects
  - Columns: `#`, URL (as clickable link), Last Modified, Change Frequency, Priority (as colored badge)
- **Footer**: "Generated dynamically" note
- **Styling**: All inline CSS (XSLT requirement) using the design tokens above
- **Fonts**: Google Fonts import for Lora, Merriweather, Rajdhani (matching `layout.tsx`)
- **Effects**: Subtle glassmorphism-inspired card container, gradient header, smooth hover transitions

### 2. Modify `src/app/sitemap.xml/route.ts`
Add the `<?xml-stylesheet?>` processing instruction to the XML output (line 103):
```xml
<?xml version="1.0" encoding="UTF-8"?>
<?xml-stylesheet type="text/xsl" href="/sitemap.xsl"?>
<urlset ...>
```

## Implementation Notes
- The XSL file goes in `public/` so it's served as a static asset at `/sitemap.xsl`
- All CSS must be inline within the XSLT (no external stylesheet reference possible for XSLT)
- Priority badges: color-coded (green for 1.0, blue for 0.8, gray for lower)
- Responsive: works on mobile with horizontal scroll on the table
- The `<xsl:output method="html">` ensures clean HTML output
